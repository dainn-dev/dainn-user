Introduction

DainnStripe is a comprehensive .NET/C# library that provides enterprise-grade Stripe integration for payments, subscriptions, webhooks, and catalog management. Built on top of Stripe.NET with Entity Framework Core for persistence and audit trails.

🎯 Purpose: DainnStripe eliminates the complexity of Stripe integration by providing pre-built services, webhook handlers, database persistence, and idempotency guarantees out of the box.

Why DainnStripe?

  • Production-Ready: Built-in webhook idempotency, retry logic, and error handling
  • Database Persistence: Automatic tracking of payments, subscriptions, and events
  • Multi-Tenant Support: Stripe Connect integration for marketplace platforms
  • Type-Safe: Strongly-typed models and services with full IntelliSense support
  • Extensible: Easy to customize and extend for your business logic
  • Database Agnostic: Supports SQL Server, PostgreSQL, MySQL, and SQLite

Target Audience

.NET developers building SaaS applications, e-commerce platforms, or marketplace solutions that require payment processing.

Key Capabilities

Installation

Prerequisites

  • .NET 8.0 SDK or later
  • Stripe Account: Sign up at stripe.com
  • Database: SQL Server, PostgreSQL, MySQL, or SQLite
  • Stripe CLI: (Optional) For local webhook testing

Install via NuGet

Install the DainnStripe package in your ASP.NET Core project:

dotnet add package DainnStripe
💡 Single Package: DainnStripe is distributed as a single NuGet package that includes all services, entities, and configurations. No need to install multiple packages.

Get Your Stripe API Keys

  1. Log in to your Stripe Dashboard
  2. Navigate to Developers → API keys
  3. Copy your Secret key (starts with sk_test_ or sk_live_)
  4. Copy your Publishable key (starts with pk_test_ or pk_live_)
⚠️ Security: Never commit API keys to source control. Use environment variables, Azure Key Vault, or user secrets for development.

Install Stripe CLI (Optional)

For local webhook testing:

# macOS
brew install stripe/stripe-cli/stripe

# Windows (with Scoop)
scoop bucket add stripe https://github.com/stripe/scoop-stripe-cli.git
scoop install stripe

# Or download from https://github.com/stripe/stripe-cli/releases

Configuration

appsettings.json Structure

Add the DainnStripe configuration section to your appsettings.json:

{
  "DainnStripe": {
    "Enabled": true,
    "SecretKey": "sk_test_your_secret_key_here",
    "PublishableKey": "pk_test_your_publishable_key_here",
    "WebhookSigningSecret": "whsec_your_webhook_secret_here",
    "WebhookToleranceSeconds": 300,
    "WebhookPath": "/stripe/webhooks",
    "ThrowOnApiVersionMismatch": false
  },
  "ConnectionStrings": {
    "DainnStripe": "Data Source=dainnstripe.db"
  }
}

Configuration Options

Option Type Required Description
Enabled bool No Enable/disable Stripe integration (default: true)
SecretKey string Yes Stripe secret API key (sk_test_* or sk_live_*)
PublishableKey string Yes Stripe publishable key (pk_test_* or pk_live_*)
WebhookSigningSecret string Yes Webhook endpoint signing secret (whsec_*)
WebhookToleranceSeconds long No Timestamp tolerance for webhook validation (default: 300)
WebhookPath string No Webhook endpoint path (default: /stripe/webhooks)
ThrowOnApiVersionMismatch bool No Reject events with API version mismatch (default: false)

Quickstart (Payments Only)

Get up and running with Stripe payments in 10 minutes.

Step 1: Create a New ASP.NET Core Web API

dotnet new webapi -n MyPaymentApp
cd MyPaymentApp

Step 2: Install DainnStripe

dotnet add package DainnStripe

Step 3: Configure appsettings.json

{
  "DainnStripe": {
    "SecretKey": "sk_test_your_secret_key",
    "PublishableKey": "pk_test_your_publishable_key",
    "WebhookSigningSecret": "whsec_your_webhook_secret"
  },
  "ConnectionStrings": {
    "DainnStripe": "Data Source=payments.db"
  }
}

Step 4: Update Program.cs

using DainnStripe;

var builder = WebApplication.CreateBuilder(args);

// Add DainnStripe services
builder.Services.AddDainnStripe(builder.Configuration);

builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();

// Add DainnStripe middleware
app.UseDainnStripe();

// Map webhook endpoint
app.MapDainnStripeWebhooks();

app.MapControllers();

app.Run();

Step 5: Run Database Migrations

dotnet ef migrations add InitialCreate
dotnet ef database update

Step 6: Create a Payment Controller

using DainnStripe.Interfaces;
using DainnStripe.Models;
using Microsoft.AspNetCore.Mvc;

[ApiController]
[Route("api/[controller]")]
public class PaymentsController : ControllerBase
{
    private readonly IDainnStripePaymentService _paymentService;

    public PaymentsController(IDainnStripePaymentService paymentService)
    {
        _paymentService = paymentService;
    }

    [HttpPost("create-payment-intent")]
    public async Task<IActionResult> CreatePaymentIntent([FromBody] CreatePaymentIntentRequest request)
    {
        var result = await _paymentService.CreatePaymentIntentAsync(request);
        return Ok(result);
    }
}

Step 7: Test the API

dotnet run

Navigate to https://localhost:5001/swagger and test the payment endpoint.

✅ Success! You now have a working Stripe payment integration with automatic database persistence and webhook handling.

Multi-Tenant Setup

For marketplace applications using Stripe Connect:

using DainnStripe;
using DainnStripe.Models;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddDainnStripe(builder.Configuration);

// Add request context accessor for tenant isolation
builder.Services.AddHttpContextAccessor();

var app = builder.Build();

// Middleware to set tenant context
app.Use(async (context, next) =>
{
    var tenantId = context.Request.Headers["X-Tenant-Id"].FirstOrDefault();
    if (!string.IsNullOrEmpty(tenantId))
    {
        var contextAccessor = context.RequestServices
            .GetRequiredService<IDainnStripeRequestContextAccessor>();
        contextAccessor.SetContext(new DainnStripeRequestContext
        {
            TenantId = tenantId
        });
    }
    await next();
});

app.UseDainnStripe();
app.MapDainnStripeWebhooks();
app.MapControllers();
app.Run();

Webhooks Setup

What are Webhooks?

Webhooks are HTTP callbacks that Stripe sends to your application when events occur (e.g., payment succeeded, subscription canceled). DainnStripe automatically handles webhook verification, idempotency, and persistence.

Local Development with Stripe CLI

For local testing, use the Stripe CLI to forward webhooks:

# Login to Stripe CLI
stripe login

# Forward webhooks to your local endpoint
stripe listen --forward-to https://localhost:5001/stripe/webhooks

# The CLI will output your webhook signing secret
# Copy it to your appsettings.json or user secrets

Production Webhook Setup

  1. Go to Stripe Dashboard → Webhooks
  2. Click Add endpoint
  3. Enter your webhook URL: https://yourdomain.com/stripe/webhooks
  4. Select events to listen to (or select "Select all events")
  5. Copy the Signing secret (starts with whsec_)
  6. Add it to your production configuration

Recommended Events to Listen To

Event Type Description
payment_intent.succeeded Payment completed successfully
payment_intent.payment_failed Payment failed
customer.subscription.created New subscription created
customer.subscription.updated Subscription modified
customer.subscription.deleted Subscription canceled
invoice.payment_succeeded Invoice paid successfully
invoice.payment_failed Invoice payment failed
checkout.session.completed Checkout session completed

How DainnStripe Handles Webhooks

  1. Signature Verification: Validates webhook signature using signing secret
  2. Idempotency Check: Prevents duplicate processing of the same event
  3. Event Persistence: Stores event in database with processing status
  4. Handler Dispatch: Routes event to appropriate handler
  5. Status Update: Marks event as processed or failed

Custom Webhook Handlers

Create custom handlers by implementing IStripeWebhookHandler:

using DainnStripe.Interfaces;
using DainnStripe.Models;
using Stripe;

public class CustomPaymentHandler : IStripeWebhookHandler
{
    public bool CanHandle(Event stripeEvent)
    {
        return stripeEvent.Type == "payment_intent.succeeded";
    }

    public async Task<StripeWebhookProcessResult> HandleAsync(Event stripeEvent)
    {
        var paymentIntent = stripeEvent.Data.Object as PaymentIntent;

        // Your custom logic here
        Console.WriteLine($"Payment succeeded: {paymentIntent.Id}");

        return StripeWebhookProcessResult.Success();
    }
}

// Register in Program.cs
builder.Services.AddScoped<IStripeWebhookHandler, CustomPaymentHandler>();

Testing Webhooks

# Send a test webhook event
stripe trigger payment_intent.succeeded

# Or send a specific event
stripe events resend evt_1234567890
⚠️ Important: Always verify webhook signatures in production. Never trust webhook data without signature verification to prevent spoofing attacks.

Troubleshooting

Common Issues

1. Webhook Signature Verification Failed

Problem: Webhooks are rejected with signature verification errors.

Solutions:
  • Verify WebhookSigningSecret matches your Stripe webhook endpoint
  • Check that the secret starts with whsec_
  • For local testing, use Stripe CLI and copy the signing secret it provides
  • Ensure webhook payload is not modified by middleware (compression, etc.)
  • Check WebhookToleranceSeconds if server time is off

2. Payment Intent Creation Fails

Problem: Getting errors when creating payment intents.

Solutions:
  • Verify SecretKey is correct and starts with sk_test_ or sk_live_
  • Check amount is in smallest currency unit (cents for USD)
  • Ensure currency code is valid (lowercase, e.g., "usd", "eur")
  • Check Stripe API logs in dashboard for detailed error messages
  • Verify your Stripe account is activated and not restricted

3. Database Migration Errors

Problem: EF Core migrations fail or database not created.

Solutions:
  • Ensure connection string is correct in appsettings.json
  • Check database server is running
  • Verify user has permissions to create database/tables
  • For SQLite: Check file path is writable
  • Run migrations with verbose logging: dotnet ef database update --verbose

4. Webhooks Not Received Locally

Problem: Stripe CLI forwards webhooks but app doesn't receive them.

Solutions:
  • Verify MapDainnStripeWebhooks() is called in Program.cs
  • Check webhook path matches: default is /stripe/webhooks
  • Ensure app is running on the port Stripe CLI is forwarding to
  • Check firewall/antivirus not blocking connections
  • Use stripe listen --print-json to see raw webhook data

5. Idempotency Issues

Problem: Same webhook event processed multiple times.

Solutions:
  • DainnStripe automatically handles idempotency via StripeWebhookEventRecords table
  • Check database connection is stable
  • Verify StripeWebhookIdempotencyService is registered
  • Check for database transaction issues or deadlocks

Performance Optimization

Production Tips:
  • Enable database connection pooling
  • Add indexes on frequently queried columns (StripePaymentIntentId, CustomerId)
  • Use async/await for all Stripe API calls
  • Implement retry logic with exponential backoff for transient failures
  • Cache product catalog locally to reduce API calls
  • Monitor webhook processing time and optimize slow handlers
  • Use Stripe's idempotency keys for payment creation

Security Best Practices

⚠️ Important Security Measures:
  • Never expose Secret Key: Keep it server-side only
  • Always verify webhooks: Use signing secret validation
  • Use HTTPS: Required for production webhooks
  • Validate amounts: Always verify payment amounts match expected values
  • Log security events: Track failed webhook verifications
  • Rotate keys: Periodically rotate API keys and webhook secrets
  • Use test mode: Test thoroughly before going live
