Introduction

DainnStripe is a comprehensive .NET/C# library that provides enterprise-grade Stripe integration for payments, subscriptions, webhooks, and catalog management. Built on top of Stripe.NET with Entity Framework Core for persistence and audit trails.

🎯 Purpose: DainnStripe eliminates the complexity of Stripe integration by providing pre-built services, webhook handlers, database persistence, and idempotency guarantees out of the box.

Why DainnStripe?

  • Production-Ready: Built-in webhook idempotency, retry logic, and error handling
  • Database Persistence: Automatic tracking of payments, subscriptions, and events
  • Multi-Tenant Support: Stripe Connect integration for marketplace platforms
  • Type-Safe: Strongly-typed models and services with full IntelliSense support
  • Extensible: Easy to customize and extend for your business logic
  • Database Agnostic: Supports SQL Server, PostgreSQL, MySQL, and SQLite

Target Audience

.NET developers building SaaS applications, e-commerce platforms, or marketplace solutions that require payment processing.

Key Capabilities

  • One-time payments with Payment Intents
  • Recurring subscriptions with automatic billing
  • Stripe Checkout Sessions for hosted payment pages
  • Webhook processing with automatic persistence
  • Product and price catalog synchronization
  • Stripe Connect for multi-tenant marketplaces
  • Money movement (transfers, payouts, balance tracking)

Installation

Prerequisites

  • .NET 8.0 SDK or later
  • Stripe Account: Sign up at stripe.com
  • Database: SQL Server, PostgreSQL, MySQL, or SQLite
  • Stripe CLI: (Optional) For local webhook testing

Install via NuGet

Install the DainnStripe package in your ASP.NET Core project:

dotnet add package DainnStripe
💡 Single Package: DainnStripe is distributed as a single NuGet package that includes all services, entities, and configurations. No need to install multiple packages.

Get Your Stripe API Keys

  1. Log in to your Stripe Dashboard
  2. Navigate to Developers → API keys
  3. Copy your Secret key (starts with sk_test_ or sk_live_)
  4. Copy your Publishable key (starts with pk_test_ or pk_live_)
⚠️ Security: Never commit API keys to source control. Use environment variables, Azure Key Vault, or user secrets for development.

Configuration

appsettings.json Structure

Add the DainnStripe configuration section to your appsettings.json:

{
  "DainnStripe": {
    "Enabled": true,
    "SecretKey": "sk_test_your_secret_key_here",
    "PublishableKey": "pk_test_your_publishable_key_here",
    "WebhookSigningSecret": "whsec_your_webhook_secret_here",
    "WebhookToleranceSeconds": 300,
    "WebhookPath": "/stripe/webhooks",
    "ThrowOnApiVersionMismatch": false,
    "Database": {
      "Provider": "SQLite",
      "ConnectionString": "Data Source=dainnstripe.db"
    }
  }
}

Configuration Options

Option Type Required Description
Enabled bool No Enable/disable Stripe integration (default: true)
SecretKey string Yes Stripe secret API key (sk_test_* or sk_live_*)
PublishableKey string Yes Stripe publishable key (pk_test_* or pk_live_*)
WebhookSigningSecret string Yes Webhook endpoint signing secret (whsec_*)
WebhookToleranceSeconds long No Timestamp tolerance for webhook validation (default: 300)
WebhookPath string No Webhook endpoint path (default: /stripe/webhooks)
ThrowOnApiVersionMismatch bool No Reject events with API version mismatch (default: false)

Quickstart (Payments Only)

Get up and running with Stripe payments in 10 minutes.

Step 1: Create a New ASP.NET Core Web API

dotnet new webapi -n MyPaymentApp
cd MyPaymentApp

Step 2: Install DainnStripe

dotnet add package DainnStripe

Step 3: Configure appsettings.json

{
  "DainnStripe": {
    "SecretKey": "sk_test_your_secret_key",
    "PublishableKey": "pk_test_your_publishable_key",
    "WebhookSigningSecret": "whsec_your_webhook_secret",
    "Database": {
      "Provider": "SQLite",
      "ConnectionString": "Data Source=payments.db"
    }
  }
}

Step 4: Update Program.cs

using DainnStripe;

var builder = WebApplication.CreateBuilder(args);

// Add DainnStripe services
builder.Services.AddDainnStripe(builder.Configuration);

builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();

// Add DainnStripe middleware
app.UseDainnStripe();

// Map webhook endpoint
app.MapDainnStripeWebhooks();

app.MapControllers();

app.Run();

Step 5: Run Database Migrations

dotnet ef migrations add InitialCreate
dotnet ef database update

Step 6: Create a Payment Controller

using DainnStripe.Interfaces;
using DainnStripe.Models;
using Microsoft.AspNetCore.Mvc;

[ApiController]
[Route("api/[controller]")]
public class PaymentsController : ControllerBase
{
    private readonly IDainnStripePaymentService _paymentService;

    public PaymentsController(IDainnStripePaymentService paymentService)
    {
        _paymentService = paymentService;
    }

    [HttpPost("create-payment-intent")]
    public async Task<IActionResult> CreatePaymentIntent([FromBody] CreatePaymentIntentRequest request)
    {
        var result = await _paymentService.CreatePaymentIntentAsync(request);
        return Ok(result);
    }
}

Step 7: Test the API

dotnet run

Navigate to https://localhost:5001/swagger and test the payment endpoint.

✅ Success! You now have a working Stripe payment integration with automatic database persistence and webhook handling.

using DainnStripe;
using DainnStripe.Models;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddDainnStripe(builder.Configuration);

// Add request context accessor for tenant isolation
builder.Services.AddHttpContextAccessor();

var app = builder.Build();

// Middleware to set tenant context
app.Use(async (context, next) =>
{
    var tenantId = context.Request.Headers["X-Tenant-Id"].FirstOrDefault();
    if (!string.IsNullOrEmpty(tenantId))
    {
        var contextAccessor = context.RequestServices
            .GetRequiredService<IDainnStripeRequestContextAccessor>();
        contextAccessor.SetContext(new DainnStripeRequestContext
        {
            TenantId = tenantId
        });
    }
    await next();
});

app.UseDainnStripe();
app.MapDainnStripeWebhooks();
app.MapControllers();
app.Run();

Webhooks Setup

What are Webhooks?

Webhooks are HTTP callbacks that Stripe sends to your application when events occur (e.g., payment succeeded, subscription canceled). DainnStripe automatically handles webhook verification, idempotency, and persistence.

Local Development with Stripe CLI

For local testing, use the Stripe CLI to forward webhooks:

# Login to Stripe CLI
stripe login

# Forward webhooks to your local endpoint
stripe listen --forward-to https://localhost:5001/stripe/webhooks

# The CLI will output your webhook signing secret
# Copy it to your appsettings.json or user secrets

Production Webhook Setup

  1. Go to Stripe Dashboard → Webhooks
  2. Click Add endpoint
  3. Enter your webhook URL: https://yourdomain.com/stripe/webhooks
  4. Select events to listen to (or select "Select all events")
  5. Copy the Signing secret (starts with whsec_)
  6. Add it to your production configuration

Recommended Events to Listen To

Event Type Description
payment_intent.succeeded Payment completed successfully
payment_intent.payment_failed Payment failed
customer.subscription.created New subscription created
customer.subscription.updated Subscription modified
customer.subscription.deleted Subscription canceled
invoice.payment_succeeded Invoice paid successfully
invoice.payment_failed Invoice payment failed
checkout.session.completed Checkout session completed

How DainnStripe Handles Webhooks

  1. Signature Verification: Validates webhook signature using signing secret
  2. Idempotency Check: Prevents duplicate processing of the same event
  3. Event Persistence: Stores event in database with processing status
  4. Handler Dispatch: Routes event to appropriate handler
  5. Status Update: Marks event as processed or failed

Custom Webhook Handlers

Create custom handlers by implementing IStripeWebhookHandler:

using DainnStripe.Entities;
using DainnStripe.Interfaces;
using Stripe;

public class CustomPaymentHandler : IStripeWebhookHandler
{
    // Return true for event types this handler should process
    public bool CanHandle(string eventType)
        => eventType == "payment_intent.succeeded";

    public async Task HandleAsync(
        Event stripeEvent,
        StripeWebhookEventRecord record,
        CancellationToken cancellationToken = default)
    {
        var paymentIntent = stripeEvent.Data.Object as PaymentIntent;

        // Your custom logic here
        Console.WriteLine($"Payment succeeded: {paymentIntent.Id}");
    }
}

// Register in Program.cs (after AddDainnStripe)
builder.Services.AddScoped<IStripeWebhookHandler, CustomPaymentHandler>();

Testing Webhooks

# Send a test webhook event
stripe trigger payment_intent.succeeded

# Or send a specific event
stripe events resend evt_1234567890
⚠️ Important: Always verify webhook signatures in production. Never trust webhook data without signature verification to prevent spoofing attacks.

Built-In Endpoint Groups

DainnStripe ships with ready-made minimal API endpoint groups you can map in Program.cs:

// Stripe webhook (required)
app.MapDainnStripeWebhooks();                       // POST /stripe/webhooks

// Commerce: checkout, payment intents, subscriptions
app.MapDainnStripeCommerceEndpoints();              // /dainnstripe/commerce/*

// Catalog: products, prices, Stripe sync
app.MapDainnStripeCatalogEndpoints();              // /dainnstripe/catalog/*

// Connect: tenants, connected accounts, onboarding links
app.MapDainnStripeConnectEndpoints();              // /dainnstripe/connect/*

// Money movement: transfers, payouts, reconciliation
app.MapDainnStripeMoneyMovementEndpoints();        // /dainnstripe/money-movement/*

All groups except the webhook require authorization by default. Override the pattern and auth requirement:

app.MapDainnStripeCatalogEndpoints(
    pattern: "/api/billing/catalog",
    requireAuthorization: false);

Catalog Management

When to use this: DainnStripe Catalog is designed for a small, managed set of items — SaaS pricing plans, membership tiers, or a handful of recurring prices. It is not intended as the primary catalog for an e-commerce store with hundreds or thousands of products. For large product catalogs see the E-commerce domain guide.

Upsert a Product

When StripeProductId is omitted and a IDainnStripeClientFactory is registered, DainnStripe automatically creates the product on Stripe and stores the returned ID locally. Pass an explicit StripeProductId to link an existing Stripe product without making an API call. 

// Auto-provision on Stripe (StripeProductId omitted)
var product = await catalogService.UpsertProductAsync(new UpsertCatalogProductRequest
{
    LookupKey   = "plan-pro",
    Name        = "Pro Plan",
    Description = "Access to all Pro features",
    Active      = true
});
// product.StripeProductId is now populated from Stripe

// Link to an existing Stripe product
var product = await catalogService.UpsertProductAsync(new UpsertCatalogProductRequest
{
    LookupKey       = "plan-pro",
    StripeProductId = "prod_abc123",   // pre-existing Stripe product
    Name            = "Pro Plan",
    Active          = true
});

Upsert a Price

When StripePriceId is omitted, DainnStripe creates the price on Stripe automatically. The parent product must already have a StripeProductId. 

// Auto-provision recurring price on Stripe
var price = await catalogService.UpsertPriceAsync(new UpsertCatalogPriceRequest
{
    ProductLookupKey = "plan-pro",
    LookupKey        = "plan-pro-monthly",
    UnitAmount       = 1900,                           // $19.00
    Currency         = "usd",
    Interval         = DainnStripePriceInterval.Month,
    IntervalCount    = 1,
    Active           = true
});
// price.StripePriceId is now populated — use it in checkout line items

// One-time price (no recurrence)
var price = await catalogService.UpsertPriceAsync(new UpsertCatalogPriceRequest
{
    ProductLookupKey = "plan-pro",
    LookupKey        = "plan-pro-onetime",
    UnitAmount       = 4900,
    Currency         = "usd",
    Interval         = DainnStripePriceInterval.None,  // one-time
    Active           = true
});

Query Active Catalog

Small catalogs — load all

// Returns all active products with active prices.
// Suitable for rendering a pricing page with a few plans.
var products = await catalogService.GetActiveCatalogAsync();

Large catalogs — paginated (recommended)

// Returns one page at a time — safe for any catalog size.
var page = await catalogService.GetActiveCatalogPagedAsync(page: 1, pageSize: 50);

Console.WriteLine($"Page {page.Page} of {page.TotalPages} ({page.TotalCount} total)");

foreach (var product in page.Items)
{
    foreach (var price in product.Prices)
        Console.WriteLine($"  {product.Name}: {price.UnitAmount} {price.Currency}");
}

// Next page
if (page.HasNextPage)
{
    var next = await catalogService.GetActiveCatalogPagedAsync(page: 2, pageSize: 50);
}

// CatalogPage properties:
// .Items          — IReadOnlyList<DainnStripeProduct> for this page
// .Page           — current 1-based page number
// .PageSize       — items per page (clamped to 1–200)
// .TotalCount     — total active products across all pages
// .TotalPages     — Math.Ceiling(TotalCount / PageSize)
// .HasNextPage    — Page < TotalPages
// .HasPreviousPage — Page > 1

Sync Catalog from Stripe

IDainnStripeCatalogSyncService.SyncFromStripeAsync pulls all active products and prices from Stripe using auto-pagination (handles any catalog size) and upserts them into the local database. Use this to seed the local catalog or reconcile after changes made directly in the Stripe Dashboard. 

// Trigger manually or from a background job / IHostedService
var result = await catalogSyncService.SyncFromStripeAsync(triggerSource: "startup");

Console.WriteLine($"Products: +{result.ProductsCreated} created, ~{result.ProductsUpdated} updated");
Console.WriteLine($"Prices:   +{result.PricesCreated} created, ~{result.PricesUpdated} updated");

// CatalogSyncResult properties:
// .SyncRunId        — Guid of the DainnStripeCatalogSyncRun audit record
// .Succeeded        — true if sync completed without exception
// .ProductsCreated  — number of new local products created
// .ProductsUpdated  — number of existing local products updated
// .PricesCreated    — number of new local prices created
// .PricesUpdated    — number of existing local prices updated

Domain: E-commerce

Use this when selling products or services with one-time payments. Two checkout mechanisms are available: Checkout Session (Stripe-hosted page — simple, redirect-based) or PaymentIntent (custom UI embedding Stripe Elements directly).

Do you need to create products on Stripe?

CreateCheckoutSessionLineItem requires a StripePriceId — a Stripe Price that already exists on your Stripe account. This does not mean you need to pre-create every product in the Stripe Dashboard.

Catalog sizeRecommended approach
Small (< 50 items) — SaaS plans, memberships Use IDainnStripeCatalogService to manage products & prices. DainnStripe auto-creates them on Stripe and stores StripePriceId locally.
Large (50 + items) — E-commerce product catalog Keep the full catalog in your own database. Add a StripePriceId column and populate it lazily on first checkout (see pattern below). Stripe stores nothing until a user checks out.

Pattern A — Large catalog: lazy StripePriceId

Your product table already has all the data you need. Add a nullable StripePriceId column. On the first checkout for each product, create a Stripe Price on demand and cache the ID back. Every subsequent checkout reuses the cached ID — no Stripe API call needed. 

// Helper: get or lazily create a Stripe Price for a product
private async Task<string> GetOrCreateStripePriceAsync(Product product, CancellationToken ct)
{
    if (!string.IsNullOrEmpty(product.StripePriceId))
        return product.StripePriceId;

    // First checkout for this product — provision on Stripe now
    var stripeProductService = new ProductService();
    var stripeProduct = await stripeProductService.CreateAsync(new ProductCreateOptions
    {
        Name        = product.Name,
        Description = product.Description
    }, cancellationToken: ct);

    var stripePriceService = new PriceService();
    var stripePrice = await stripePriceService.CreateAsync(new PriceCreateOptions
    {
        Product    = stripeProduct.Id,
        UnitAmount = product.PriceCents,
        Currency   = "usd"
    }, cancellationToken: ct);

    // Cache back to your DB so future checkouts skip the Stripe call
    product.StripePriceId = stripePrice.Id;
    await _db.SaveChangesAsync(ct);

    return stripePrice.Id;
}

// Checkout endpoint
app.MapPost("/checkout", async (CartDto cart, IDainnStripeCheckoutService checkoutService,
    HttpContext httpContext, AppDbContext db, CancellationToken ct) =>
{
    var request = new CreateCheckoutSessionRequest
    {
        OwnerId    = httpContext.User.FindFirst("sub")!.Value,
        Mode       = DainnStripeCheckoutMode.Payment,
        SuccessUrl = "https://yourshop.com/order/success?session={CHECKOUT_SESSION_ID}",
        CancelUrl  = "https://yourshop.com/cart"
    };

    foreach (var item in cart.Items)
    {
        var product = await db.Products.FindAsync(item.ProductId);
        var stripePriceId = await GetOrCreateStripePriceAsync(product!, ct);

        request.LineItems.Add(new CreateCheckoutSessionLineItem
        {
            StripePriceId = stripePriceId,
            Quantity      = item.Quantity,
            UnitAmount    = product!.PriceCents,
            Currency      = "usd"
        });
    }

    request.Metadata["order_id"] = cart.OrderId.ToString();

    var result = await checkoutService.CreateAsync(request, ct);
    // DainnStripe persists DainnStripePayment { Status = Pending }

    return Results.Redirect(result.Url!);
});
Stripe stores nothing until the first checkout for each product. You are not pre-creating 1 000 products on Stripe — only products that have ever been purchased will appear in your Stripe Dashboard.

public class MarketplacePaymentWebhookHandler : IStripeWebhookHandler
{
    private readonly IDainnStripeMoneyMovementService _moneyMovement;

    public MarketplacePaymentWebhookHandler(IDainnStripeMoneyMovementService moneyMovement)
        => _moneyMovement = moneyMovement;

    public bool CanHandle(string eventType)
        => eventType == "checkout.session.completed";

    public async Task HandleAsync(
        Event stripeEvent,
        StripeWebhookEventRecord record,
        CancellationToken cancellationToken = default)
    {
        var session = (Session)stripeEvent.Data.Object;

        if (!session.Metadata.TryGetValue("seller_id", out var sellerId))
            return;

        // Transfer 90% to seller, retain 10% as platform fee
        var sellerAmount = (long)(session.AmountTotal!.Value * 0.9);

        await _moneyMovement.CreateTransferAsync(new CreateTransferRequest
        {
            OwnerId                    = sellerId,
            StripeDestinationAccountId = sellerStripeAccountId,
            Amount                     = sellerAmount,
            Currency                   = "usd",
            TransferGroup              = session.Metadata["order_id"]
        }, cancellationToken);
    }
}

Querying Invoices from the Local Database

// Inject DainnStripeDbContext for direct queries
public class InvoiceController : ControllerBase
{
    private readonly DainnStripeDbContext _db;

    public InvoiceController(DainnStripeDbContext db) => _db = db;

    [HttpGet("api/invoices")]
    public async Task<IActionResult> GetInvoices(CancellationToken ct)
    {
        var invoices = await _db.DainnStripeInvoices
            .Where(i => i.OwnerId == User.FindFirst("sub")!.Value)
            .OrderByDescending(i => i.CreatedAt)
            .Select(i => new
            {
                i.StripeInvoiceId,
                i.AmountDue,
                i.AmountPaid,
                i.Currency,
                i.Status,
                i.HostedInvoiceUrl,
                i.InvoicePdfUrl,
                i.PeriodStart,
                i.PeriodEnd
            })
            .ToListAsync(ct);

        return Ok(invoices);
    }
}

Troubleshooting

Common Issues

1. Webhook Signature Verification Failed

Problem: Webhooks are rejected with signature verification errors.

Solutions:
  • Verify WebhookSigningSecret matches your Stripe webhook endpoint
  • Check that the secret starts with whsec_
  • For local testing, use Stripe CLI and copy the signing secret it provides
  • Ensure webhook payload is not modified by middleware (compression, etc.)
  • Check WebhookToleranceSeconds if server time is off

2. Payment Intent Creation Fails

Problem: Getting errors when creating payment intents.

Solutions:
  • Verify SecretKey is correct and starts with sk_test_ or sk_live_
  • Check amount is in smallest currency unit (cents for USD)
  • Ensure currency code is valid (lowercase, e.g., "usd", "eur")
  • Check Stripe API logs in dashboard for detailed error messages
  • Verify your Stripe account is activated and not restricted

3. Database Migration Errors

Problem: EF Core migrations fail or database not created.

Solutions:
  • Ensure connection string is correct in appsettings.json
  • Check database server is running
  • Verify user has permissions to create database/tables
  • For SQLite: Check file path is writable
  • Run migrations with verbose logging: dotnet ef database update --verbose

4. Webhooks Not Received Locally

Problem: Stripe CLI forwards webhooks but app doesn't receive them.

Solutions:
  • Verify MapDainnStripeWebhooks() is called in Program.cs
  • Check webhook path matches: default is /stripe/webhooks
  • Ensure app is running on the port Stripe CLI is forwarding to
  • Check firewall/antivirus not blocking connections
  • Use stripe listen --print-json to see raw webhook data

5. Idempotency Issues

Problem: Same webhook event processed multiple times.

Solutions:
  • DainnStripe automatically handles idempotency via StripeWebhookEventRecords table
  • Check database connection is stable
  • Verify StripeWebhookIdempotencyService is registered
  • Check for database transaction issues or deadlocks

Performance Optimization

Production Tips:
  • Enable database connection pooling
  • Add indexes on frequently queried columns (StripePaymentIntentId, CustomerId)
  • Use async/await for all Stripe API calls
  • Implement retry logic with exponential backoff for transient failures
  • Cache product catalog locally to reduce API calls
  • Monitor webhook processing time and optimize slow handlers
  • Use Stripe's idempotency keys for payment creation

Security Best Practices

⚠️ Important Security Measures:
  • Never expose Secret Key: Keep it server-side only
  • Always verify webhooks: Use signing secret validation
  • Use HTTPS: Required for production webhooks
  • Validate amounts: Always verify payment amounts match expected values
  • Log security events: Track failed webhook verifications
  • Rotate keys: Periodically rotate API keys and webhook secrets
  • Use test mode: Test thoroughly before going live
